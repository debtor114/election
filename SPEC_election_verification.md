# 6·3 지방선거 개표 데이터 검증 배터리 — 작업 명세 (Claude Code용)

## 목표
선관위 공개 개표 데이터로, 떠도는 의혹/우려를 **재현 가능한 통계 검정**으로 판정한다.
모든 검정은 (관측값) + (우연 기대치/기준선)을 함께 출력해, "이상인가"를 σ나 임계값으로
답한다. 결론을 하드코딩하지 말 것 — 숫자와 임계값을 찍고 PASS(정상)/FLAG(정밀조사)만 표시.

## 데이터 (T0 먼저)
- 출처: 선관위 선거통계시스템(info.nec.go.kr) → "읍면동별 개표결과" 엑셀.
- 대상 선거: 시도지사 17개(우선), 가능하면 교육감·기초단체장도.
- 파싱 결과 tidy 스키마:
  `[시도, 시군구, 읍면동, 구분(관내사전/관외사전/선거일/계), <후보별 득표...>, 계, 투표수, 무효]`
- NEC 엑셀 특유의 병합셀/다중헤더/소계행을 정리. 후보 열은 race마다 다르므로 설정으로 분리.
- 산출: `data/<race>.parquet` + 로드 함수.

## 검정 (표준 개표 데이터만으로 가능)
### T1 — 동일 득표 전수 검정  [출발점: check_duplicate_votes.py]
- 기준별 동일 쌍 집계: ① 한 후보 단독 동일 ② 상위 2후보 동시 동일 ③ 전 후보+총계 동일(복제).
- 우연 기대치: 각 읍면동 총표는 고정, 득표율을 경험분포에서 부트스트랩 재추출 → null 분포 → 관측값이 몇 σ.
- 방향성 검정: 동일 쌍이 특정 정당으로 쏠리는지(이항검정).
- FLAG 조건: 관측이 null +3σ 초과 / 방향 +3σ 쏠림 / 복제 쌍 ≥ 1.

### T2 — 사전 vs 본투표 득표율 격차의 '일정함'  [출발점: early_vs_dayof.py]
- 후보별 사전·본 득표율, 격차의 평균·표준편차.
- 회귀 `사전 ~ 본`의 R²·잔차편차. 정상 데이터는 흩어짐이 큼.
- FLAG 조건: R²≈1 & 잔차≈0 (부자연스럽게 완벽한 직선).

### T3 — 투표율 × 승자 득표율 포렌식 (Shpilkin류)
- 읍면동별 (투표율, 승자 득표율) 산점도 + 2D 히스토그램(plot 저장).
- 고투표율·고득표율 구석의 비정상 집중(투표함 채우기 시그니처) 정량화.
- FLAG 조건: 고투표율 꼬리에서 승자 득표율이 비정상적으로 상승하는 군집 검출.

### T4 — 끝자리/벤포드 (보조, 약함)
- 후보별 득표수 끝자리 균일성(카이제곱), 벤포드 1st-digit.
- ⚠ 득표수엔 신뢰도 낮음 — 단독 근거 금지, 보조 지표로만 보고.

## 검정 (부족 투표소 목록 추가 필요 — `data/shortage_list.csv`)
> 목록은 선관위 발표·언론·국정조사 자료에서 수집. 컬럼: `[시도,시군구,읍면동,투표소]`.

### T5 — 부족 투표소의 정치 편향 (순열검정)
- 부족 투표소들의 보수(또는 특정 정당) 득표율 평균 vs 전체에서 동수 무작위표본 분포.
- FLAG 조건: 관측 평균이 순열분포의 양측 99.7%(±3σ) 밖.

### T6 — 부족 = 당일 투표율 초과 여부
- 읍면동별 당일 투표율 계산. 부족 투표소가 '당일 투표율 상위 꼬리'에 몰리는지 검정.
- 정상 투표율인데 부족했던 투표소를 별도 리스트로 출력(= 과실 설명으로 설명 안 되는 이상치).

## 산출물
- `tests/T1..T6.py` (각 독립 실행) + `run_all.py`(요약 리포트: 검정별 PASS/FLAG + 수치 근거).
- `README.md`: 데이터 받는 법, 컬럼 매핑 설정법, 한계(T4 약함, T5/T6는 목록 필요).
- 모든 결과는 표/플롯 파일로 저장. 콘솔엔 판정 한 줄씩.

## 원칙
1. 관측값과 기준선을 항상 함께. "수상하다" 금지, "관측 X, 기대 Y±Z, = Nσ"로.
2. 컬럼/후보/정당 매핑은 설정 파일로 분리(race·연도마다 다름).
3. 결론 문구 하드코딩 금지. 데이터가 말하게 할 것.
