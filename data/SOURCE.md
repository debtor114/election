# 데이터 출처·스냅샷·검증

이 폴더의 데이터는 "내 파일을 믿어라"가 아니라 **"공식 출처에서 그대로 다시 만들어 내고,
아무것도 안 건드렸음을 확인할 수 있다"** 를 목표로 구성했습니다.

## 구성
```
data/
├── raw/                     ← 손대지 않은 원본 (선관위 표 그대로, 후보 컬럼 wide)
│   ├── <선거종류>.csv.gz       (7종)
│   └── CHECKSUMS.sha256       ← 원본 '비압축 내용'의 SHA-256
├── processed/
│   └── nec_2026_local.csv.gz  ← 가공본 (long/tidy, 분석·검정용)
├── past_elections/            ← 대조군: 2018·2022 지방선거 (출처 다름, 아래 참고)
│   ├── ...제7회...20180613.xlsx · ...제8회...20220601.xlsx
│   └── CHECKSUMS.sha256 · SOURCE.txt
├── shortage_list.example.csv  ← T5·T6용 부족 투표소 목록 양식
└── SOURCE.md (이 파일)
```

> **2018·2022 출처(중요):** 이 두 해는 info.nec.go.kr 선거통계시스템(현재 선거만 서빙)에 **없습니다.**
> **공공데이터포털 data.go.kr**의 "중앙선거관리위원회 제7·8회 전국동시지방선거 개표결과" 공식 xlsx에서
> 받았습니다(`data/past_elections/`). 2026(위 raw/processed)은 info.nec.go.kr, 2018·2022는 data.go.kr — 출처가 다릅니다.

- **raw = 원본**: `nec_download.py`가 선관위에서 받은 그대로. 숫자 미가공.
- **processed = 가공본**: `make_dataset.py`로 *형태만* long으로 바꾼 것(숫자 불변).
- raw·gz는 용량 때문에 압축했을 뿐이며, 체크섬은 *비압축 내용* 기준입니다(압축 타임스탬프 영향 배제).

## 스스로 검증하는 법

**(A) 가공본이 원본을 안 건드렸나** — 한 줄로:
```bash
python verify_dataset.py
# → raw 를 똑같이 변환해 processed 와 1:1 대조. 행수·총득표수·셀단위 일치 확인.
```

**(B) 원본 자체가 선관위와 같은가** — 직접 다시 받아 대조:
```bash
python nec_download.py --type all --out nec_data_check
# 받은 nec_data_check/<선거종류>/_combined.csv 의 SHA-256을
# data/raw/CHECKSUMS.sha256 과 비교 (값 일치가 핵심; 라이브러리 버전에 따라
# 바이트 해시가 미세하게 다를 수 있어, 값 비교는 verify_dataset 방식 권장).
```

## 한계·주의 (정직하게)
- **스냅샷 고정**: 위 수집일 기준입니다. 선관위 원천이 추후 갱신·정정돼도 이 스냅샷으로 결과가 재현됩니다.
  *원천이 바뀌면 (B)의 재다운로드 결과는 스냅샷과 다를 수 있습니다.*
- **개인정보 없음**: 읍면동/투표종류별 집계치로, 개인 식별 정보가 없습니다.
- **재배포**: 선관위 공개 데이터의 가공본입니다. 재사용 시 출처(선관위)를 밝히고 선관위/공공데이터
  이용약관(공공누리 등)을 확인하세요. 약관상 통째 재배포가 걸리면, 데이터 대신
  `nec_download.py` + `CHECKSUMS.sha256` 만으로도 동일한 투명성을 얻을 수 있습니다(각자 원천에서 수집).
- 이 저장소는 **검증 도구이지 평결이 아닙니다.** 출력은 대부분 단서·반증이며, 해석은 읽는 사람의 몫입니다.
